# OMV APIサーバー 要件定義

## 概要

OMV (Open Media Vault) NAS上で動作する軽量APIサーバー。MacアプリおよびiPhoneアプリからのリクエストを処理し、写真・動画ファイルの受信・配信を担う。

## 機能要件

### FR-API-01: ファイル受信 (アップロード)

- Macアプリからの写真・動画ファイルを受信し、NASの指定ディレクトリに保存する
- UDP/非同期転送プロトコルをサポートし、大容量ファイルの高速転送を実現する
- 分割送信されたファイルを受信・結合する
- チェックサムで転送整合性を検証する

### FR-API-02: ファイル配信 (ダウンロード)

- iPhoneアプリからのサムネイル・フル解像度画像・動画のリクエストに応答する
- サムネイルはサーバーサイドで生成・キャッシュする
- Range リクエスト (部分取得) をサポートする

### FR-API-03: 認証・認可

- Auth0 JWT トークンの検証を行う
- ロールベースのアクセス制御 (Mac: 読み書き、iPhone: 読み取りのみ)
- シークレットは AWS Parameter Store から取得する。クレデンシャルは `/tmp/aws/credential.json` から読み込み、有効期限切れ時に自動リフレッシュする

### FR-API-04: メタデータプロキシ

- iPhoneアプリからのSupabaseクエリを中継する (iPhoneはSupabaseに直接アクセスしない)
- 自然言語クエリをClaude Haikuで SQL に変換し実行する

### FR-API-05: ヘルスチェック

- `/health` エンドポイントで稼働確認
- 接続中クライアント数を返す

## 非機能要件

### NFR-API-01: パフォーマンス
- 同時接続数: 通常1、最大2を想定した設計でよい
- 大容量ファイル (数GBの動画) を効率的に処理できること

### NFR-API-02: 軽量性
- OMVのNASリソース (低スペックCPU) で動作できること
- Go によるシングルバイナリで配置・管理を容易にする

### NFR-API-03: 信頼性
- サーバー再起動時の自動復旧 (systemdサービス)
- 転送中断時の再開機能

## テスト要件

- **ユニットテスト**: JWT検証、ファイル分割結合、チェックサム検証
- **プロパティテスト** (gopter/testing/quick):
  - 任意のチャンクサイズで分割・結合が冪等であること
  - 任意のJWTペイロードで検証が正しく成功/失敗すること
  - 任意のファイルパスでディレクトリトラバーサルが防がれること
- **統合テスト**: エンドポイントごとのHTTPテスト
