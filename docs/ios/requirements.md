# iPhoneアプリ 要件定義

## 概要

iPhoneで動作するネイティブビューアーアプリ。OMV NAS上の写真・動画をサムネイルキャッシュを活用してストレスなく閲覧でき、キーワードや自然言語で高速検索できる。

## 機能要件

### FR-IOS-01: 写真・動画一覧表示

- グリッド形式でサムネイル一覧を表示する
- 撮影日・アルバム・場所などでグルーピング・フィルタリングができる
- サムネイルは積極的にキャッシュし、スクロールがスムーズであること (60fps 維持)

### FR-IOS-02: 詳細表示

- フル解像度表示 (ピンチズーム対応)
- 動画再生 (シークバー付き)
- EXIFメタデータ表示 (撮影日時・場所・カメラ情報)
- AIタグ表示 (オブジェクト・シーン)

### FR-IOS-03: キーワード検索

- タグ・場所・日付範囲・カメラ機種などでフィルタリング
- 検索結果をリアルタイムで更新

### FR-IOS-04: 自然言語検索

- 「去年の夏の海の写真」「子供が映っている動画」などの自然言語クエリを入力できる
- OMV APIサーバーを経由してClaude Haikuで SQL に変換・実行し結果を表示する

### FR-IOS-05: 認証

- Auth0によるログイン
- ログイン状態の永続化 (Keychain)
- OMV APIサーバーへのトークン付きリクエスト

### FR-IOS-06: オフライン対応

- サムネイルキャッシュはオフライン時も利用可能
- オフライン時は検索・詳細表示を制限し、適切なメッセージを表示する

## 非機能要件

### NFR-IOS-01: パフォーマンス
- サムネイル表示: 100枚のグリッドスクロールで60fps維持
- 検索結果: クエリ入力から1秒以内に初期結果を表示

### NFR-IOS-02: セキュリティ
- AWSへの直接アクセスは行わない
- **Supabaseへの直接アクセスは行わない** (Supabaseの接続情報をiPhoneに持たせない)
- 全通信はAuth0トークン認証済みのOMV APIサーバー経由

## テスト要件

- **ユニットテスト**: キャッシュ管理、Auth0トークン処理、APIクライアント
- **プロパティテスト** (SwiftCheck):
  - 任意の検索クエリでアプリがクラッシュしないこと
  - キャッシュのLRU削除ポリシーが正しく動作すること
  - サムネイルキャッシュが容量制限を超えないこと
- **UIテスト**: 主要な画面遷移のSnapshot/XCUITest
