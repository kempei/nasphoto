# データベース 要件定義

## 概要

Supabaseの無料枠で稼働するPostgreSQLデータベース。写真・動画のメタデータを保存し、iPhoneアプリからの高速検索をサポートする。

## 機能要件

### FR-DB-01: メタデータ保存

以下のメタデータを保存する:
- **基本情報**: ファイルID (UUID)、ファイル名、NAS上のパス、ファイルサイズ、MIMEタイプ、SHA256ハッシュ
- **撮影情報**: 撮影日時 (EXIF)、Photos.app取り込み日時
- **カメラ情報**: カメラメーカー・機種、レンズ情報、露出/絞り/ISO/焦点距離
- **位置情報**: 緯度・経度、国・都市・場所名 (リバースジオコーディング)
- **AIタグ**: オブジェクトタグ (配列)、シーンタグ (配列)、顔クラスターID (配列)
- **サムネイル**: サムネイルURL (NAS APIサーバーへのパス)
- **管理情報**: バックアップ日時、メタデータ更新日時

### FR-DB-02: 検索インデックス

- 撮影日時でのソート・フィルタリング
- AIタグ (配列) での全文検索 (GINインデックス)
- 位置情報での地理的検索 (PostGIS)
- カメラ機種でのフィルタリング

### FR-DB-03: 無料枠制約内での運用

- Supabase無料枠: 500MB DB、50,000行/月 API呼び出し
- サムネイルバイナリはSupabaseに保存せず、NASサーバーのURLを参照する
- 1レコードのサイズを最小化する

## テスト要件

- **スキーマテスト**: マイグレーションが冪等であること
- **プロパティテスト**:
  - 任意の有効なメタデータが正しく挿入・取得できること
  - 全文検索インデックスが正しく機能すること
- **パフォーマンステスト**: 10万件のデータで検索クエリが1秒以内に返ること
