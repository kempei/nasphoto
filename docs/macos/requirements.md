# NasPhotos.app 要件定義

## 概要

Mac mini 上で常駐する独自のネイティブバックグラウンドアプリ (**NasPhotos.app**)。

iPhone・カメラからの写真・動画の取り込みは既存の **写真.app (Photos.app)** が担う。NasPhotos.app は写真.app に取り込まれた写真・動画を PhotosKit 経由で読み取り、Vision・Core ML 等の Apple SDK でメタデータを解析・補完する。ファイル本体は OMV API Server 経由で OMV NAS へ転送し、解析済みメタデータは Supabase へ直接保存する。

## 機能要件

### FR-MAC-01: 自動バックアップ

- Photos.appに取り込まれた写真・動画を自動検出し、OMV NASへバックアップする
- バックアップ済みかどうかの判定は「Photos.appへの取り込み日時」を基準とする (全件スキャン不要)
- Mac起動中はバックグラウンドで継続的に動作する
- バックアップ失敗時はリトライ処理を行う

### FR-MAC-02: 転送プロトコル

- OMV APIサーバーのカスタムプロトコル (UDP/非同期) を使用して高速転送する
- 大容量ファイル (動画) は分割転送に対応する

### FR-MAC-03: メタデータ解析

- **EXIFメタデータ**: 撮影日時、GPS座標、カメラ機種、レンズ情報、露出設定など
- **AI解析 (Vision framework / Core ML)**:
  - 含まれるオブジェクト認識 (人物、動物、建物、乗り物など)
  - シーン分類 (屋内/屋外、自然/都市など)
  - 顔認識・人物クラスタリング
- ローカルGPU (Apple Silicon Neural Engine) を活用してオンデバイス処理

### FR-MAC-04: スキャン写真のメタデータ補完

- 紙写真をスキャンした場合など、EXIFが欠損している場合:
  - ユーザーとのインタラクティブな対話で撮影日時・場所などを入力
  - 前後の写真のメタデータから撮影日時・場所を類推して補完候補を提示
  - ユーザーが確認・承認後にメタデータを確定

### FR-MAC-05: メタデータ保存

- 解析済みメタデータをSupabase (クラウドDB) に保存する
- 保存する情報: ファイルパス (NAS上)、ハッシュ値、撮影日時、GPS、カメラ情報、AIタグ、サムネイルURL

### FR-MAC-06: UI

- メニューバーアプリとして動作 (常駐)
- バックアップ進捗・最終バックアップ日時を表示
- エラー通知
- 設定画面: OMV接続先、Auth0認証情報、除外フォルダ設定

## 非機能要件

### NFR-MAC-01: パフォーマンス
- 通常使用中 (写真撮影・編集) と並行して動作し、CPU使用率が著しく高くなってはならない
- バックアップは取り込みから5分以内に開始する

### NFR-MAC-02: 信頼性
- macOSの再起動後も自動で起動・再開する
- ネットワーク断絶後の自動再接続

### NFR-MAC-03: セキュリティ
- Auth0によるOMV APIサーバーへの認証
- AWS Parameter StoreからAPIキー等を取得 (macOSデフォルトのAWS認証情報を使用)
- ローカルキャッシュの暗号化

## テスト要件

- **ユニットテスト**: メタデータパーサー、バックアップ状態管理、リトライロジック
- **プロパティテスト**: 
  - 任意のEXIFデータが与えられてもパーサーがクラッシュしないこと
  - バックアップ済み判定ロジックが日時の順序関係を正しく保持すること
  - メタデータ補完ロジックが有効な候補のみ返すこと
- **統合テスト**: OMV APIサーバーのモックに対してバックアップフロー全体を確認
